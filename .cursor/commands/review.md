# 代码审查

审查当前改动（未提交变更或指定文件/PR），给出可执行反馈。

## 输入

用户可在命令后指定：文件、PR、关注点（性能/安全/可读性/数据正确性）。未指定则审查工作区全部变更。

## 要求

1. 先给结论：可合并 / 需小改 / 需重做。
2. 按严重程度列出问题：正确性 → 数据一致性 → 回归风险 → 风格。
3. 每条问题附上文件路径与修改建议（尽量给到具体改法）。
4. 对本仓库重点检查：
   - 看板：周期、KPI、图表与表格是否一致；是否破坏离线/部署假设。
   - README 列表：表格对齐、badge、分类与计数。
5. 不改代码，除非用户明确要求“顺便修”。
6. 用简体中文，简洁分点。
