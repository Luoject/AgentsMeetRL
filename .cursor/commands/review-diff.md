# 审查当前改动

对当前工作区或指定分支的 diff 做一次聚焦代码审查。

## 步骤

1. 查看 `git status` 与相关 diff（优先未提交改动；若干净则看相对 `main` 的分支 diff）。
2. 按严重程度指出问题：正确性、数据一致性（README 表格 / badge / 看板数字）、破坏性格式变更、死链、明显回归。
3. 区分「必须修」与「可选建议」；不要为风格吹毛求疵。
4. 若涉及 `knowledge-ops-dashboard.html`，重点检查周期文案、KPI 与图表 data 是否一致。
5. 若涉及 `README.md` 新仓库行，检查链接、徽章 URL、分类是否合理。

## 输出

用简洁中文给出：结论（可合并 / 需修改）、问题列表、建议下一步。
