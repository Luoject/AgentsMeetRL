# 更新知识底座运维周报看板

根据用户提供的本周数据，更新 `knowledge-ops-dashboard.html`。

## 输入

用户会在命令后附上：报告周期（如 `2026-07-06 ~ 2026-07-12`）、KPI、表格、图表数据或原始周报文本。若信息不全，先列出缺失项，再用合理占位并标注 `TODO`。

## 要求

1. 只改看板相关文件（优先 `knowledge-ops-dashboard.html`；若存在离线看板/邮件模板/配置，一并同步周期与标题）。
2. 保持现有视觉风格、分区结构与 Chart.js 用法，不重构整页。
3. 同步更新：页面 `<title>`、页头周期徽章、周次、摘要 KPI、各分区表格与图表数据、页脚日期。
4. 数字格式与现网一致（万/亿、百分比、千分位等）。
5. 改完后自检：关键 KPI、Top 列表、明细表、图表 dataset 是否与输入一致。
6. 用简体中文回复变更摘要；提交信息用中文约定式，例如 `feat: 刷新看板数据至 YYYY-MM-DD ~ YYYY-MM-DD 周报`。
