# 更新运维周报看板

刷新 `knowledge-ops-dashboard.html` 为新一周的知识底座运维运营周报。

## 输入

用户提供本周数据，通常包括：统计周期、KPI（语料总量、入网篇数、搜索次数等）、Top 知识组、失败原因、搜索应用/命中、资源监控（EC2 / MongoDB / MySQL 等）、问题与改进事项、MaaS 模型调用明细。

若用户只给部分数据，只更新有依据的部分，并在回复里标明未更新的区块。

## 步骤

1. 读取当前 `knowledge-ops-dashboard.html`，定位标题中的周期、KPI 卡片、图表数据、表格与问题列表。
2. 将 `<title>`、页眉周期文案更新为用户给出的新周期（格式：`YYYY-MM-DD ~ YYYY-MM-DD`）。
3. 按用户数据更新 KPI、TopN、失败原因、搜索、资源监控、问题改进、MaaS 明细等；保持现有布局、CSS 变量与 Chart.js 用法，不要重做视觉设计。
4. 检查图表 `data` 数组与图例标签是否与新数字一致，避免文案与图脱节。
5. 本地用静态服务打开页面做一次冒烟确认（能打开、主区块可见即可）。

## 约束

- 不要编造监控数字；缺数据就保留旧值或留空，并明确告知。
- 不要引入新的前端框架或拆成多文件；继续保持单文件看板。
- 改动范围尽量限制在数据与必要文案。

## 输出

用几句话总结：新周期、关键 KPI 变化、改动文件。
